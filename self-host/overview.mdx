# Self-Hosting c15t Backend

c15t provides a powerful, self-hostable consent management backend that you can deploy on your own infrastructure. This gives you complete control over your data, compliance requirements, and deployment environment.

## Why Self-Host?

- **Data Sovereignty**: Keep all consent data on your own infrastructure
- **Compliance**: Meet specific regulatory requirements for data residency
- **Customization**: Full control over configuration and deployment
- **Privacy**: No third-party data processing
- **Cost Control**: Optimize infrastructure costs for your scale

## Architecture

The c15t backend is built with:

- **Language**: TypeScript/Node.js
- **Runtime**: Node.js 18.17.0+ or Bun
- **API Framework**: oRPC with OpenAPI support
- **Database Support**: Multiple adapters for flexibility
- **Deployment**: Framework-agnostic (works with Express, Next.js, Bun, etc.)

## Available Versions

### Version 2 (Current)

The latest version with improved architecture and FumaDB adapters:

```typescript
import { c15tInstance } from '@c15t/backend/v2';
```

- Modern FumaDB adapter system
- Simplified configuration
- Better TypeScript support
- Enhanced performance

### Version 1 (Legacy)

The original version, now deprecated:

```typescript
import { c15tInstance } from '@c15t/backend';
```

## Features

### Consent Management
- Track and manage user consent preferences
- Support for GDPR, CCPA, LGPD compliance
- Version-controlled consent policies
- Audit logging for all consent events

### Geo-Location
- Automatic detection of user jurisdiction
- Show region-specific consent banners
- Configurable geo-location services

### Multi-Domain Support
- Manage consent across multiple domains
- Subdomain handling
- Domain-specific policies

### Server-Side Translation
- Reduce banner bundle size
- 40+ languages supported
- Custom translation overrides

### Webhooks & Integrations
- Real-time consent event webhooks
- Integration with analytics platforms
- Custom event handlers
