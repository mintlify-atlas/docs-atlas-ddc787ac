# Iframe Blocking

The c15t JavaScript SDK includes an iframe blocker that prevents third-party iframes from loading until users grant the necessary consent. This is essential for GDPR compliance when embedding content from services like YouTube, Google Maps, or social media platforms.

## Quick Start

The iframe blocker is automatically initialized when you create a consent manager store:

```typescript
import { createConsentManagerStore, configureConsentManager } from 'c15t';

const manager = configureConsentManager({
  backendURL: '/api/c15t',
});

const store = createConsentManagerStore(manager, {
  iframeBlockerConfig: {
    disableAutomaticBlocking: false, // Enable automatic iframe blocking
  },
});
```

## How It Works

The iframe blocker:
1. **Monitors** all iframes on the page using a MutationObserver
2. **Checks** each iframe for a `data-category` attribute
3. **Blocks** iframes if the required consent hasn't been granted
4. **Unblocks** iframes automatically when consent is granted

## Marking Iframes for Blocking

Add a `data-category` attribute to iframes that require consent:

```html
<!-- YouTube embed - requires marketing consent -->
<iframe 
  data-category="marketing"
  data-src="https://www.youtube.com/embed/VIDEO_ID"
  width="560" 
  height="315"
></iframe>

<!-- Google Maps - requires functionality consent -->
<iframe 
  data-category="functionality"
  data-src="https://www.google.com/maps/embed?pb=..."
  width="600" 
  height="450"
></iframe>

<!-- Twitter embed - requires marketing consent -->
<iframe
  data-category="marketing"
  data-src="https://platform.twitter.com/embed/..."
  width="500"
  height="600"
></iframe>
```

<Note>
  Use `data-src` instead of `src` for consent-gated iframes. The blocker will move `data-src` to `src` when consent is granted.
</Note>

## Consent Categories

Valid consent categories match the c15t consent types:

- `necessary` - Essential functionality
- `functionality` - Enhanced features
- `measurement` - Analytics and metrics
- `marketing` - Advertising and tracking
- `experience` - Personalization

```html
<!-- Analytics iframe -->
<iframe 
  data-category="measurement"
  data-src="https://analytics-dashboard.example.com"
></iframe>

<!-- Social media widget -->
<iframe 
  data-category="marketing"
  data-src="https://facebook.com/plugins/..."
></iframe>
```

## Programmatic Control

Manually control iframe blocking using the store methods:

### Process All Iframes

Manually process all iframes based on current consent state:

```typescript
// Process iframes manually
store.getState().processIframes();
```

### Update Consent State

The iframe blocker automatically reacts to consent changes:

```typescript
// Grant marketing consent - YouTube iframes will automatically unblock
await store.getState().saveConsents('all');

// Revoke marketing consent - YouTube iframes will be blocked again
store.getState().setConsent('marketing', false);
await store.getState().saveConsents('custom');
```

## Direct API Usage

For advanced use cases, you can use the iframe blocker API directly:

```typescript
import { createIframeBlocker } from 'c15t';

// Create iframe blocker with initial consent state
const blocker = createIframeBlocker(
  {
    disableAutomaticBlocking: false,
  },
  {
    necessary: true,
    functionality: false,
    measurement: false,
    marketing: false,
    experience: false,
  }
);

// Update consent state
blocker.updateConsents({
  marketing: true, // Unblock marketing iframes
});

// Manually process iframes
blocker.processIframes();

// Clean up when done
blocker.destroy();
```

See API at `src/libs/iframe-blocker/core.ts`

## IframeBlocker Interface

```typescript
interface IframeBlocker {
  /**
   * Updates the consents and processes iframes accordingly
   */
  updateConsents: (newConsents: Partial<ConsentState>) => void;

  /**
   * Processes all iframes on the page based on current consent state
   */
  processIframes: () => void;

  /**
   * Destroys the iframe blocker and restores original behavior
   */
  destroy: () => void;
}
```

## Behavior

### When Consent is Granted

1. The `data-src` attribute is moved to `src`
2. The iframe loads the content
3. The `data-src` attribute is removed

### When Consent is Revoked

1. The `src` attribute is cleared
2. The original URL is preserved (if it was in `data-src`)
3. The iframe content is unloaded

### Without Category Attribute

Iframes without a `data-category` attribute are not managed by the blocker and load normally.

```html
<!-- This iframe always loads -->
<iframe src="https://example.com"></iframe>
```

See test example at `src/libs/iframe-blocker/__tests__/core.test.ts:23`

## Error Handling

The iframe blocker throws an error for invalid consent categories:

```html
<!-- Invalid category -->
<iframe data-category="invalid-category" data-src="..."></iframe>
```

```typescript

// Valid categories: necessary, functionality, measurement, marketing, experience
```

## Best Practices

<Tip>
  - Always use `data-src` instead of `src` for consent-gated iframes
  - Use appropriate consent categories based on the iframe's purpose
  - Provide clear UI feedback when iframes are blocked
  - Test iframe blocking with different consent states
</Tip>

<Warning>
  Iframes without a `data-category` attribute will always load. Make sure to add the attribute to any iframe that requires consent.
</Warning>

## Integration with Script Loader

Combine iframe blocking with the script loader for comprehensive consent management:

```typescript
const store = createConsentManagerStore(manager, {
  // Configure iframe blocking
  iframeBlockerConfig: {
    disableAutomaticBlocking: false,
  },
  
  // Configure script loading
  scripts: [
    {
      id: 'youtube-api',
      src: 'https://www.youtube.com/iframe_api',
      category: 'marketing',
      onLoad: () => {
        console.log('YouTube API loaded - iframes can now use the API');
      },
    },
  ],
});
```

Both the script loader and iframe blocker will automatically respond to consent changes.
