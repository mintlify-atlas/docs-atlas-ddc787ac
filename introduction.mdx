## What is c15t?

c15t is a headless consent engine that transforms privacy management from a compliance checkbox into a fully observable system. Built for modern development teams, it provides a flexible, performant solution for managing user consent across web applications.

## Why c15t?

### Developer-first design

Built by developers for developers. c15t provides a clean, type-safe API that integrates seamlessly with your existing toolchain.

```typescript
import { createConsentManagerStore } from 'c15t';

const store = createConsentManagerStore(manager, {
  scripts: [
    {
      id: 'google-analytics',
      src: 'https://www.googletagmanager.com/gtag/js?id=GA_MEASUREMENT_ID',
      category: 'measurement',
      onLoad: () => {
        window.dataLayer = window.dataLayer || [];
        window.gtag('consent', 'update', { analytics_storage: 'granted' });
      }
    }
  ]
});
```

### Framework agnostic

Works across JavaScript frameworks with first-class support for React, Next.js, and vanilla JavaScript.

<CodeGroup>
```tsx React
import { ConsentManagerProvider, CookieBanner } from '@c15t/react';

function App() {
  return (
    <ConsentManagerProvider options={{ mode: 'offline' }}>
      <YourApp />
      <CookieBanner />
    </ConsentManagerProvider>
  );
}
```

```tsx Next.js
import { ConsentManagerProvider, CookieBanner } from '@c15t/react';

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <ConsentManagerProvider options={{ mode: 'offline' }}>
          {children}
          <CookieBanner />
        </ConsentManagerProvider>
      </body>
    </html>
  );
}
```

```javascript JavaScript
import { configureConsentManager, createConsentManagerStore } from 'c15t';

const manager = configureConsentManager({ mode: 'offline' });
const store = createConsentManagerStore(manager);
```
</CodeGroup>

### GDPR compliant by default

Automatic compliance with GDPR, CCPA, LGPD, and other privacy regulations. c15t handles consent categories, geo-location detection, and audit logging out of the box.

<Note>
  c15t provides **five consent categories** aligned with GDPR requirements: necessary, functionality, measurement, experience, and marketing.
</Note>

### Server-side rendering support

Full SSR support for Next.js and other frameworks. Consent state syncs seamlessly between server and client.

### Fully observable

Track every consent interaction with built-in callbacks and event hooks:

```typescript
const store = createConsentManagerStore(manager, {
  callbacks: {
    onConsentSet: ({ preferences }) => {
      console.log('Consent updated:', preferences);
      analytics.track('consent_updated', preferences);
    },
    onBannerFetched: ({ showConsentBanner, jurisdiction }) => {
      console.log('Banner decision:', showConsentBanner, jurisdiction);
    }
  }
});
```

## Key features

<AccordionGroup>
  <Accordion title="Consent categories" icon="list-check">
    Five built-in consent categories aligned with GDPR:
    
    - **Necessary**: Essential cookies required for basic functionality (always enabled)
    - **Functionality**: Enhanced features and personalization
    - **Measurement**: Analytics and performance tracking
    - **Experience**: User experience improvements
    - **Marketing**: Advertising and marketing purposes
  </Accordion>
  
  <Accordion title="Script loader" icon="code">
    Dynamically load third-party scripts based on user consent:
    
    ```typescript
    const store = createConsentManagerStore(manager, {
      scripts: [
        {
          id: 'google-analytics',
          src: 'https://www.googletagmanager.com/gtag/js?id=GA_ID',
          category: 'measurement'
        },
        {
          id: 'facebook-pixel',
          src: 'https://connect.facebook.net/en_US/fbevents.js',
          category: 'marketing'
        }
      ]
    });
    ```
    
    Scripts automatically load when consent is granted and unload when revoked.
  </Accordion>
  
  <Accordion title="Internationalization" icon="globe">
    Built-in translation support for multiple languages:
    
    - Automatic browser language detection
    - Server-side translation when using c15t backend
    - Custom translation configs
    - Partial translation support
  </Accordion>
  
  <Accordion title="Customizable UI" icon="palette">
    Headless components with full styling control:
    
    - Pre-built, accessible components
    - Compound component pattern for custom layouts
    - Theme system with CSS variables
    - Tailwind CSS support
  </Accordion>
  
  <Accordion title="Geo-location detection" icon="map-location-dot">
    Automatically detect user location and apply appropriate regulations:
    
    - GDPR for EU users
    - CCPA for California residents
    - Other regional privacy laws
    - Jurisdiction-aware banner display
  </Accordion>
</AccordionGroup>

## Use cases

<Steps>
  <Step title="SaaS applications">
    Implement compliant consent management for your SaaS product with minimal setup. c15t handles consent state, geo-location detection, and script loading automatically.
  </Step>
  
  <Step title="Marketing websites">
    Load marketing and analytics scripts only when users consent. Integrate with Google Tag Manager, Meta Pixel, PostHog, and other popular tools.
  </Step>
  
  <Step title="E-commerce platforms">
    Ensure GDPR and CCPA compliance for your online store while maintaining conversion tracking and marketing capabilities.
  </Step>
  
  <Step title="Content platforms">
    Manage consent for embedded content like YouTube videos, social media widgets, and third-party integrations.
  </Step>
</Steps>
