## Overview

The `Frame` component provides consent-gated content rendering for embedded content such as iframes, videos, and third-party widgets. It acts as a consent boundary, conditionally rendering children based on user consent and displaying a placeholder when permission is not granted.

<Note>
  Children are not mounted until consent is given, preventing unnecessary network requests and ensuring GDPR compliance.
</Note>

## Import

```tsx
import { Frame } from '@c15t/react';
```

## Basic Usage

```tsx
import { Frame } from '@c15t/react';

function VideoEmbed() {
  return (
    <Frame category="marketing">
      <iframe 
        src="https://www.youtube.com/embed/VIDEO_ID"
        width="560"
        height="315"
        title="YouTube video"
      />
    </Frame>
  );
}
```

## Props

### FrameProps

```typescript
interface FrameProps extends ComponentPropsWithRef<'div'> {
  children: ReactNode;
  category: AllConsentNames;
  placeholder?: ReactNode;
  noStyle?: boolean;
  theme?: FrameTheme;
}
```

<ParamField path="children" type="ReactNode" required>
  Content rendered when consent is granted. Children are not mounted until consent is given, preventing unnecessary network requests.
</ParamField>

<ParamField path="category" type="AllConsentNames" required>
  Consent category required to render children. Must be one of:
  - `'necessary'`
  - `'functionality'`
  - `'experience'`
  - `'measurement'`
  - `'marketing'`
</ParamField>

<ParamField path="placeholder" type="ReactNode">
  A custom placeholder component to display when consent is not met. If not provided, a default placeholder with a consent button will be displayed.
</ParamField>

<ParamField path="noStyle" type="boolean" default="false">
  When true, removes all default styling from the component.
</ParamField>

<ParamField path="theme" type="FrameTheme">
  Custom theme to override default styles while maintaining structure and accessibility. Merges with defaults. Ignored when `noStyle={true}`.
  
  See [FrameTheme](#frametheme) for available customization options.
</ParamField>

<ParamField path="...props" type="ComponentPropsWithRef<'div'>">
  All other props are forwarded to the root `div` element, including `className`, `style`, and `ref`.
</ParamField>

## Compound Components

The `Frame` component includes compound components for building custom placeholders:

### Frame.Root

Root container for custom placeholders.

```tsx
import { Frame } from '@c15t/react';

<Frame category="marketing" placeholder={
  <Frame.Root>
    {/* Custom placeholder content */}
  </Frame.Root>
}>
  <iframe src="..." />
</Frame>
```

### Frame.Title

Title element for custom placeholders.

```tsx
import { Frame } from '@c15t/react';

<Frame category="marketing" placeholder={
  <Frame.Root>
    <Frame.Title>Custom Title</Frame.Title>
  </Frame.Root>
}>
  <iframe src="..." />
</Frame>
```

### Frame.Button

Consent button for custom placeholders.

```tsx
import { Frame } from '@c15t/react';

<Frame category="marketing" placeholder={
  <Frame.Root>
    <Frame.Title>Enable Marketing Content</Frame.Title>
    <Frame.Button category="marketing">
      Allow Marketing Cookies
    </Frame.Button>
  </Frame.Root>
}>
  <iframe src="..." />
</Frame>
```

## Theme Customization

### FrameTheme

Configuration object for styling different parts of the Frame component.

```typescript
type FrameTheme = Partial<{
  'frame.placeholder.root': ThemeValue<PlaceholderCSSVariables>;
  'frame.placeholder.title': ThemeValue;
  'frame.placeholder.button': ThemeValue<ButtonCSSVariables>;
}>;
```

<ParamField path="frame.placeholder.root" type="ThemeValue<PlaceholderCSSVariables>">
  Styles for the placeholder root container element.
  
  **Available CSS variables:**
  - `--frame-placeholder-gap`
  - `--frame-placeholder-border-radius`
  - `--frame-placeholder-border-width`
  - `--frame-placeholder-border-color`
  - `--frame-placeholder-border-color-dark`
  - `--frame-placeholder-background-color`
  - `--frame-placeholder-background-color-dark`
  - `--frame-placeholder-text-color`
  - `--frame-placeholder-text-color-dark`
  - `--frame-placeholder-title-color`
  - `--frame-placeholder-title-color-dark`
  - `--frame-placeholder-shadow`
  - `--frame-placeholder-shadow-dark`
  - `--frame-placeholder-opacity`
  - `--frame-placeholder-animation`
  - `--frame-font-family`
  - `--frame-line-height`
</ParamField>

<ParamField path="frame.placeholder.title" type="ThemeValue">
  Styles for the placeholder title element.

<ParamField path="frame.placeholder.button" type="ThemeValue<ButtonCSSVariables>">
  Styles for the placeholder button element. See button component documentation for available CSS variables.
</ParamField>

### Multiple Categories

For content requiring multiple consent categories, use the core `has` utility with complex conditions:

```tsx
import { Frame } from '@c15t/react';
import { useConsentManager } from '@c15t/react';

function AdvancedTracking() {
  const { has } = useConsentManager();
  
  // Check for complex conditions
  const hasRequiredConsent = has({ and: ['measurement', 'marketing'] });
  
  if (!hasRequiredConsent) {
    return <div>Requires both measurement and marketing consent</div>;
  }
  
  return (
    <iframe src="https://advanced-tracking.example.com" />
  );
}
```

## Behavior

### Mounting Strategy

- **No consent**: Placeholder is shown, children are **not mounted**
- **Consent granted**: Children are mounted, placeholder is removed
- **Consent revoked**: Children are **unmounted**, placeholder is shown again

This ensures:
- No network requests are made until consent is granted
- Scripts in children don't execute without consent
- Resources are freed when consent is revoked

### FOUC Prevention

The component prevents Flash of Unstyled Content (FOUC) by:
1. Rendering nothing until styles are loaded
2. Using `requestAnimationFrame` to ensure CSS is applied
3. Maintaining accessibility during the loading phase

### Accessibility

- Forwards `ref` to root element
- Supports all standard `div` attributes
- Default placeholder includes semantic button for consent
- Maintains focus management when switching between placeholder and content
