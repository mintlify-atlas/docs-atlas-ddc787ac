## Overview

The `migrate` command runs database migrations for self-hosted c15t backends. It reads your backend configuration, connects to your database, and applies the necessary schema changes.

## Usage

```bash
c15t self-host migrate
```

## Prerequisites

1. **Backend Configuration**: You must have a `c15t-backend.config.ts` file in your project.
2. **Database Connection**: Your database must be accessible and credentials configured.
3. **Dependencies**: Required database adapter packages must be installed.

## What It Does

1. **Locates Backend Config**: Finds or creates `c15t-backend.config.ts`
2. **Installs Dependencies**: Installs required database adapter packages if needed
3. **Loads Configuration**: Loads your database configuration
4. **Connects to Database**: Establishes connection to your database
5. **Runs Migrations**: Applies schema migrations
6. **Generates ORM Files**: Creates ORM-specific files if applicable (Prisma, Drizzle)

## Configuration File

### Required: `c15t-backend.config.ts`

```typescript
import { defineBackendConfig } from '@c15t/backend';

export const backendConfig = defineBackendConfig({
  database: {
    type: 'postgres',
    connectionString: process.env.DATABASE_URL,
  },
});
```

## Database Schema

The migration creates the following tables:

### Core Tables

<AccordionGroup>
  <Accordion title="subjects">
    Stores user/subject information.
    
    ```sql
    CREATE TABLE subjects (
      id UUID PRIMARY KEY,
      external_id VARCHAR(255),
      identity_provider VARCHAR(100),
      created_at TIMESTAMP DEFAULT NOW(),
      updated_at TIMESTAMP DEFAULT NOW()
    );
    
    CREATE INDEX idx_subjects_external_id ON subjects(external_id);
    ```
  </Accordion>
  
  <Accordion title="domains">
    Stores registered domains.
    
    ```sql
    CREATE TABLE domains (
      id UUID PRIMARY KEY,
      name VARCHAR(255) UNIQUE NOT NULL,
      created_at TIMESTAMP DEFAULT NOW()
    );
    ```
  </Accordion>
  
  <Accordion title="consent_policies">
    Stores consent policies.
    
    ```sql
    CREATE TABLE consent_policies (
      id UUID PRIMARY KEY,
      domain_id UUID REFERENCES domains(id),
      type VARCHAR(50) NOT NULL,
      version VARCHAR(50),
      is_active BOOLEAN DEFAULT true,
      created_at TIMESTAMP DEFAULT NOW()
    );
    
    CREATE INDEX idx_policies_domain_type ON consent_policies(domain_id, type);
    ```
  </Accordion>
  
  <Accordion title="consents">
    Main consent records.
    
    ```sql
    CREATE TABLE consents (
      id UUID PRIMARY KEY,
      subject_id UUID REFERENCES subjects(id),
      domain_id UUID REFERENCES domains(id),
      policy_id UUID REFERENCES consent_policies(id),
      status VARCHAR(50) DEFAULT 'active',
      given_at TIMESTAMP DEFAULT NOW(),
      revoked_at TIMESTAMP,
      metadata JSONB
    );
    
    CREATE INDEX idx_consents_subject ON consents(subject_id);
    CREATE INDEX idx_consents_domain ON consents(domain_id);
    CREATE INDEX idx_consents_policy ON consents(policy_id);
    ```
  </Accordion>
  
  <Accordion title="consent_records">
    Historical consent records.
    
    ```sql
    CREATE TABLE consent_records (
      id UUID PRIMARY KEY,
      consent_id UUID REFERENCES consents(id),
      action VARCHAR(50) NOT NULL,
      recorded_at TIMESTAMP DEFAULT NOW(),
      metadata JSONB
    );
    
    CREATE INDEX idx_records_consent ON consent_records(consent_id);
    ```
  </Accordion>
  
  <Accordion title="consent_purposes">
    Specific consent purposes (e.g., marketing, analytics).
    
    ```sql
    CREATE TABLE consent_purposes (
      id UUID PRIMARY KEY,
      record_id UUID REFERENCES consent_records(id),
      purpose_code VARCHAR(100) NOT NULL,
      consented BOOLEAN NOT NULL,
      created_at TIMESTAMP DEFAULT NOW()
    );
    
    CREATE INDEX idx_purposes_record ON consent_purposes(record_id);
    ```
  </Accordion>
  
  <Accordion title="audit_logs">
    Audit trail for all consent operations.
    
    ```sql
    CREATE TABLE audit_logs (
      id UUID PRIMARY KEY,
      consent_id UUID REFERENCES consents(id),
      action VARCHAR(100) NOT NULL,
      actor_id VARCHAR(255),
      metadata JSONB,
      created_at TIMESTAMP DEFAULT NOW()
    );
    
    CREATE INDEX idx_audit_consent ON audit_logs(consent_id);
    CREATE INDEX idx_audit_created ON audit_logs(created_at);
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Config File Not Found

```bash
✗ No backend config found.
? Would you like to create c15t-backend.config.ts? (Y/n) › Yes
```

The CLI will guide you through creating the config file.

### Database Connection Failed

```bash
✗ Failed to connect to database
Error: Connection refused at localhost:5432
```

**Solutions:**
1. Check database is running: `psql -U user -d dbname`
2. Verify connection string in `.env`
3. Check firewall/network settings
4. Ensure database exists

### Missing Dependencies

```bash
⠹ Installing required dependencies...
✓ Installed: pg, @types/pg
```

The CLI automatically installs missing database adapters.

### Migration Already Applied

```bash
✓ Database schema is already up to date.
No migrations to run.
```

This is normal - migrations are idempotent.

### Permission Errors

```bash
✗ Permission denied: Cannot create table
```

**Solution:** Ensure your database user has CREATE permissions:

```sql
GRANT CREATE ON DATABASE your_db TO your_user;
```

## Best Practices

### 1. Use Environment Variables

```bash
# .env.local
DATABASE_URL=postgresql://user:pass@localhost:5432/c15t
```

```typescript
// c15t-backend.config.ts
export const backendConfig = defineBackendConfig({
  database: {
    type: 'postgres',
    connectionString: process.env.DATABASE_URL,
  },
});
```

### 2. Run Migrations in CI/CD

```yaml
# .github/workflows/deploy.yml
steps:
  - name: Run migrations
    run: bun run c15t self-host migrate -y
    env:
      DATABASE_URL: ${{ secrets.DATABASE_URL }}
```

### 3. Backup Before Migrating

```bash
# PostgreSQL
pg_dump -U user dbname > backup.sql

# MySQL
mysqldump -u user -p dbname > backup.sql

# SQLite
cp c15t.db c15t.db.backup
```

### 4. Test Migrations Locally First

```bash
# Use a local/dev database first
DATABASE_URL=postgresql://localhost/c15t_dev c15t self-host migrate
```
