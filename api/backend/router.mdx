## Overview

The router defines all API endpoints and their implementations using oRPC (OpenAPI RPC) for type-safe, contract-based API development.

## Router Structure

```typescript
import { router } from '@c15t/backend';

// Router structure
const router = {
  consent: {
    post: postConsentHandler,
    showBanner: showConsentBannerHandler,
    verify: verifyConsentHandler,
    identify: identifyUserHandler,
  },
  meta: {
    status: statusHandler,
  },
};
```

## Direct Router Usage

You can use the router directly for type-safe API calls:

```typescript
import { c15tInstance } from '@c15t/backend';
import { kyselyAdapter } from '@c15t/backend/adapters/kysely';
import { db } from './db';

const instance = c15tInstance({
  adapter: kyselyAdapter(db),
  trustedOrigins: ['*'],
});

// Access router for type inference
const { router } = instance;

// Use with oRPC client
import { createClient } from '@orpc/client';

const client = createClient({
  baseURL: 'http://localhost:3000/api/c15t',
  router: router,
});

// Type-safe API calls
const result = await client.consent.post({
  domain: 'example.com',
  type: 'cookie_banner',
  preferences: {
    necessary: true,
    measurement: true,
  },
});
```

## Error Handling

All endpoints use standardized error responses:

```typescript
type ErrorResponse = {
  code: string;
  message: string;
  status: number;
  data?: Record<string, unknown>;
  defined: boolean;
};
```

### Common Error Codes

- `INPUT_VALIDATION_FAILED` (422): Invalid input parameters
- `SUBJECT_NOT_FOUND` (404): Subject does not exist
- `DOMAIN_NOT_FOUND` (404): Domain does not exist
- `POLICY_NOT_FOUND` (404): Policy does not exist
- `CONSENT_CREATION_FAILED` (500): Failed to create consent record
- `INTERNAL_SERVER_ERROR` (500): Unexpected server error

## Custom Error Handling

```typescript
try {
  const response = await instance.handler(request);
  
  if (!response.ok) {
    const error = await response.json();
    console.error('API Error:', error.code, error.message);
    
    // Handle specific errors
    if (error.code === 'SUBJECT_NOT_FOUND') {
      // Create new subject
    }
  }
} catch (error) {
  console.error('Request failed:', error);
}
```

## Middleware

The router automatically applies the following middleware:

- **CORS**: Cross-origin request handling
- **IP Detection**: Extracts client IP from headers
- **Telemetry**: Request tracking and tracing
- **Error Handling**: Standardized error responses

## OpenAPI Integration

The router automatically generates OpenAPI specs:

```typescript
const spec = await instance.getOpenAPISpec();

// Access spec details
console.log(spec.info.title);
console.log(spec.paths);
console.log(spec.components.schemas);
```
